FrequentlyAsked Questions

the difference between var, let, and const
  • var: Declares variables with function or global scope and allows re-declaration and updates within the same scope.
  • let: Declares variables with block scope, allowing updates but not re-declaration within the same block.
  • const: Declares block-scoped variables that cannot be reassigned after their initial assignment.
the difference between map(), forEach(), and filter()
  • map() will return a new array as per the conditions applied.
  • forEach() will not return anything. forEach() returns undefined.
  • filter()method will return an array of matching elements else will return an empty array if no matching happens.
explain arrow functions and how they are different from regular functions
  • Arrow functions are well-suited for concise, simple functions, especially as callbacks in array methods (like map, filter, reduce) or in contexts where you want to preserve the this context of the surrounding scope.
  • Regular functions are necessary when you need a dynamic this binding, when defining methods within classes, or when creating constructor functions.
how JavaScript Promises work
A JavaScript Promise is an object that represents a value that will be available in the future — it can be pending, fulfilled, or rejected — used to handle asynchronous operations cleanly.
how closures work in JavaScript
A closure in JavaScript happens when an inner function remembers and accesses variables from its outer function’s scope, even after the outer function has finished executing.
